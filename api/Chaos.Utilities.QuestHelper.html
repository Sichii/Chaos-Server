Namespace Chaos.Utilities.QuestHelper

Classes

DialogQuestHandler

Records one quest's reaction to a specific dialog template key on a specific lifecycle phase. Operations are evaluated in order; each returns true to continue the chain, false to halt (e.g., a guard failed).

Quest

Non-generic base — what the registry stores and what dispatchers reference.

QuestArg

Static factory for building a QuestArg<T> from a per-execution resolver. Pair with the implicit conversion on QuestArg<T> so builder methods can take a single QuestArg<T> parameter and accept both constants and resolvers.

QuestBuilder<TStage>

Fluent root passed to Quest<TStage>.Configure. Collects DialogQuestHandlers (and, in v2, monster/item/map/reactor handlers).

QuestContext

Non-generic base for QuestContext so framework collections (handlers, operations) can hold contexts of mixed quest types.

QuestContext<TStage>

Per-execution context passed to predicates and Run callbacks. Exposes read-only state (Source, Subject, current stage) and predicates that mirror the fluent builder's guards. All mutations live on QuestStepBuilder<TStage> — Run callbacks that need state changes should add a fluent builder method or write through Source directly.

QuestStepBuilder<TStage>

Fluent builder for an operation chain. Recorded operations are functions over QuestContext returning bool — true = continue the chain, false = halt (guard failed).

Quest<TStage>

Strongly-typed quest base. Each quest has a primary stage enum stored in Trackers.Enums.

Structs

QuestArg<T>

Wraps a value of T that may be supplied as a constant or as a resolver evaluated per quest-step execution. Methods on QuestStepBuilder<TStage> accept QuestArg<T> so call sites can pass either a literal (lifted via the implicit conversion) or a resolver via From<T>(Func<QuestContext, T>) when the value depends on per-execution state (random rolls, counters derived from the current Aisling, etc.).

Enums

DialogPhase

Identifies which dialog lifecycle event a quest handler is bound to. Mirrors IDialogScript's four overrides. Quests register handlers per phase via QuestBuilder.OnDisplaying / OnDisplayed / OnNext / OnPrevious.
